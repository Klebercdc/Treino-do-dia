2. LÓGICA DE RESOLUÇÃO DE URL — DUAL-MODE
================================================================================

Os três callers usam a mesma lógica:

  SE process.env.EXAM_OCR_SERVICE_URL está setada:
    → chama URL externa (serviço dedicado separado)
  SENÃO:
    → fallback: chama https://kronia.app.br/api/exam_ocr
       (ou seja, a função Python dentro do MESMO deploy Vercel)

4. O QUE ESTÁ QUEBRADO AGORA (root cause real)
================================================================================

O api/exam_ocr.py ESTÁ incluído no deploy (via .vercelignore com !api/exam_ocr.py).
O requirements.txt ESTÁ incluído no deploy (via !requirements.txt).
O Vercel autodetectou isso como projeto FastAPI e setou "framework: fastapi".
Com esse framework, o Vercel ignora buildCommand/outputDirectory e tenta
rodar o projeto inteiro como FastAPI — falhando porque não acha entrypoint padrão.

O vercel.json NUNCA declarou o runtime Python para api/exam_ocr.py:
  "functions": {
    "api/*.js": { "maxDuration": 60 }    ← só JS; exam_ocr.py não declarado
  }

Resultado: a função Python existe no filesystem do deploy mas não tem runtime
declarado → o Vercel não sabe executá-la → e ao mesmo tempo detectou o projeto
inteiro como FastAPI pelo requirements.txt + app = FastAPI() na raiz de api/.

================================================================================
5. CORREÇÃO — O QUE PRECISA MUDAR
================================================================================

ARQUIVO 1: vercel.json — adicionar runtime Python para exam_ocr.py

  ANTES (chave "functions" atual):
    "functions": {
      "api/*.js": { "maxDuration": 60 }
    }

  DEPOIS:
    "functions": {
      "api/*.js": { "maxDuration": 60 },
      "api/exam_ocr.py": { "runtime": "@vercel/python", "maxDuration": 60 }
    }

  Justificativa: com runtime explícito declarado, o Vercel não precisa adivinhar
  o framework pelo conteúdo dos arquivos — ele sabe que exam_ocr.py é uma função
  Python e o restante do projeto é JS + static.

ARQUIVO 2: .vercelignore — sem alteração
  As exceções !api/exam_ocr.py, !services/exam_ocr/**, !requirements.txt ficam.
  O exam_ocr e suas dependências precisam estar no deploy.

PAINEL VERCEL: trocar Framework Preset de "FastAPI" para "Other"
  Com "Other", o Vercel respeita buildCommand e outputDirectory do vercel.json
  sem tentar ser mais esperto que o repo.

  NÃO setar "Next.js" — o package.json não tem "next" como dependência e o
  build command não usa "next build". Setar Next.js vai falhar.

1.1 exam_date fallback no lab-report-orchestrator
─────────────────────────────────────────────────
Problema: quando o serviço OCR Python não retorna exam_date,
o campo ficava NULL permanentemente no banco.

Solução implementada em index.ts (feature branch → main):

  function extractExamDateFromRawText(rawText)
    Regex conservadora: Data da Coleta / Data do Exame /
    Realizado em / Coleta — sem "Data de nascimento".
    Retorna null se houver datas distintas (ambiguidade).

  function resolveExamDate(ocr)
    Prioridade: ocr.exam_date → raw_text → null
    Log sempre: [orchestrator] exam_date_source: ocr|raw_text|none

  Usada em: buildNormalizedPayload, finalizeNeedsReview,
            finalizeAnalyzed (3 pontos de persistência).

1.4 Build script — typecheck separado
──────────────────────────────────────
Problema: build e lint dependiam exclusivamente de
tsconfig.labs.json, ignorando erros de tipo fora do domínio
de labs (src/lib/supabase, src/lib/utils, etc.).

Solução em package.json:
  typecheck:web  = tsc --noEmit           (root tsconfig)
  typecheck:labs = tsc --project tsconfig.labs.json
  typecheck      = typecheck:web && typecheck:labs
  lint           = typecheck
  build          = typecheck && esbuild && scripts

  npm run build: passou sem erros

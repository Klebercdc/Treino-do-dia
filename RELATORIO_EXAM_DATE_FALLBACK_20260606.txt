1. PROBLEMA RESOLVIDO
------------------------------------------------------------

Pendência identificada no relatório anterior (seção 6-a):

  "O serviço OCR Python (services/exam_ocr) atualmente NÃO extrai a
   data de coleta do laudo — por isso os exames chegavam com
   exam_date NULL."

O orchestrator já gravava `exam_date: ocr.exam_date || null`, mas se
o OCR não retornasse a data (comportamento atual do serviço Python),
o campo ficava NULL para todos os uploads futuros.


------------------------------------------------------------
2. SOLUÇÃO IMPLEMENTADA
------------------------------------------------------------

Arquivo: supabase/functions/lab-report-orchestrator/index.ts

Duas novas funções adicionadas:

  brDateToIso(br: string): string | null
    Converte "DD/MM/YYYY" → "YYYY-MM-DD".
    Rejeita datas implausíveis (antes de 2000, além do próximo ano).

  extractExamDateFromRawText(rawText): string | null
    Extrai data de coleta de forma conservadora do raw_text do OCR.

    Regex única que cobre todos os rótulos de coleta inequívocos:
      - "Data da Coleta"
      - "Data de Coleta"
      - "Data do Exame"
      - "Realizado em"
      - "Coleta" (genérico)

    DELIBERADAMENTE AUSENTE: "Data de nascimento", "Data Entrada",
    "Emissão", "Assinatura" — todos passíveis de capturar data de
    nascimento em laudos UNIMED (problema documentado no relatório
    anterior de backfill).

    Regra de ambiguidade: se o raw_text contiver mais de uma data
    DISTINTA para esses rótulos → retorna null.
    Só grava quando há exatamente uma data candidata.

  resolveExamDate(ocr: OcrResponse): string | null
    Aplica a prioridade final:
      1. ocr.exam_date  (vem direto do OCR Python — mais confiável)
      2. extractExamDateFromRawText(ocr.raw_text)  (fallback)
      3. null  (se nenhum encontrar)
    Loga a fonte usada ("ocr" | "raw_text" | "none") para
    rastreabilidade nos logs da edge function.

Pontos de aplicação (foram de `ocr.exam_date || null` para
`resolveExamDate(ocr)`):
  - buildNormalizedPayload()   — exam_date dentro do JSON do payload
  - finalizeNeedsReview()      — exam_date na coluna do banco
  - finalizeAnalyzed()         — exam_date na coluna do banco

------------------------------------------------------------

Cenário C — raw_text ausente ou não tem rótulo de coleta:
  null. Idêntico ao comportamento anterior.

4. BUGS E PROBLEMAS ATIVOS
───────────────────────────

P0 — CRÍTICO (em produção, bloqueando feature)
──────────────────────────────────────────────

4.1  503 SERVICE_UNAVAILABLE em /api/kronia/exercises/details
     Arquivo: src/lib/exercises/local-catalog.ts (linha 160)
     Commit causador: 4243ed3 / fd5788d (2026-05-03)

     Erro no log Vercel (requestId dvsww-1777832799471):
       "Failed to load the ES module: /var/task/src/lib/exercises/local-catalog.js.
        Make sure to set "type": "module" or use .mjs extension."

     Código quebrado (linha 158-165):
       try {
         raw = JSON.parse(fs.readFileSync(LOCAL_CATALOG_FILE, 'utf8'));
       } catch {
         try {
           const alt = path.join(
             path.dirname(new URL(import.meta.url).pathname),  ← CAUSA RAIZ
             '../../../data/exercises.json'
           );
           raw = JSON.parse(fs.readFileSync(alt, 'utf8'));
         } catch {
           cachedIndex = [];
           return cachedIndex;
         }
       }

     Por que falha:
       1. import.meta.url só existe em ESM
       2. Vercel compila TypeScript para CommonJS
       3. tsconfig falta noEmitOnError → arquivo inválido é emitido
       4. Node.js 22 rejeita o módulo ao carregar
       5. Toda a rota trava em 503

     Erro TypeScript (detectado no build, ignorado):
       TS1470: The 'import.meta' meta-property is not allowed in files
       which will build into CommonJS output.

4.2  Tipo incorreto em LocalCatalogMatch
     Arquivo: src/lib/exercises/local-catalog.ts (linha 22 e 204)

     Erro TypeScript:
       TS2322: Type 'string | null' is not assignable to type 'string'.

     Causa: gifUrl declarado como string (não-nulo) mas
     sanitizeMediaUrl retorna null para CDN inativo.
     Linha 22: gifUrl: string  ← deveria ser string | null
     Linha 204: gifUrl: record.gifUrl  ← atribuição quebra tipo


P1 — ALTO
──────────

4.3  Falta noEmitOnError no tsconfig.json
     Arquivo: tsconfig.json
     Impacto: TypeScript emite JS com erros → código inválido chega a produção

4.4  Tipos ausentes em devDependencies (@types/node, @playwright/test)
     Impacto: Warnings de tipo no build, pode bloquear type-check

4.5  Lógica de plano duplicada (JS + SQL)
     api/_plans.js vs supabase/migrations/008_plan_access_and_usage.sql
     Risco: Drift de comportamento entre JS e SQL


P2 — MÉDIO
───────────

4.6  Trial duration em múltiplas fontes
     Node.js usa env var TRIAL_DAYS, SQL hardcoda 7
     Risco: Inconsistência se TRIAL_DAYS mudar

4.7  CDN inexistente referenciado em data/exercises.json
     Todas as 1300+ entradas têm gifUrl: "https://cdn.treino-do-dia.app/..."
     CDN não resolve (host não existe)
     Mitigação: cdn.treino-do-dia.app adicionado a INVALID_MEDIA_HOSTS

4.8  Feature usage logs sem idempotência forte
     Risco: Duplicidade de logs em reprocessamento


P3 — BAIXO
───────────

     Todo UI + state + utils em um arquivo
     Dificulta manutenção e debugging

4.10 index.html grande (229 KB, 7.4k linhas)
     Anormal — provavelmente contém CSS/JS inline

4.11 Cache name do sw.js com data hardcoded

5. FLUXO OFICIAL APÓS AS MUDANÇAS
------------------------------------------------------------

Usuário clica em EXAMES (Home ou Perfil)
  ↓
openLabsScreen()
  ↓
#labsScreen abre (upload + histórico)
  ↓
Usuário envia PDF/imagem
  ↓
POST /api/kronia/labs/init-upload → signed URL
  ↓
Upload direto para Storage (bucket: lab-reports)
  ↓
POST /api/kronia/labs/register → status: uploaded
  ↓
Dispatch para Edge Function lab-report-orchestrator
  ↓
OCR extrai raw_text + biomarcadores
  ↓
Clinical rules aplicadas (labRules.ts)
  ↓
Salvo em lab_reports (exam_date, ai_insights, normalized_payload)
     e lab_report_biomarkers
  ↓
Status final: analyzed
  ↓
Lista ordenada por exam_date DESC
  ↓
KRONOS lê via Context Hub (exam_date como critério primário)


------------------------------------------------------------
6. KRONOS E CONTEXT HUB
------------------------------------------------------------

- Context Hub busca o exame mais recente pela data real de
  coleta (exam_date), não pela data de upload
- Se exam_date for NULL (exames antigos), fallback para
  processed_at e depois created_at
- KRONOS recebe: data real do exame, biomarcadores, flags
  clínicas, scores de saúde, perfil do usuário (bodybuilder/beginner)
